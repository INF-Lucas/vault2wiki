# Feedback Loops in Learning Systems

Feedback loops accelerate improvement when a system can observe outcomes, compare them against intent, and make the next iteration cheaper and smarter. In this workflow, [[Feedback Loops]] and [[Local Retrieval]] are reinforcing concepts.

Useful properties of a strong learning loop:

- fast feedback
- visible metrics
- low cost of trying the next idea
- compact artifacts that can be searched later

In a personal knowledge system, a good loop often looks like this:

1. capture raw material quickly
2. compile it into a cleaner note
3. ask retrieval-based questions across the compiled notes
4. notice gaps and feed better source material back into the system

The key idea is not perfect note-taking. It is creating a compounding loop between collection, distillation, and reuse. It also suggests concept pages such as [[Compounding Knowledge]].
